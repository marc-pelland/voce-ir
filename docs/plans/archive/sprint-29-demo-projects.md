# Sprint 29 — Demo Projects

**Status:** Planned
**Goal:** Build 3 complete projects entirely via conversation to prove the end-to-end workflow, capture screen recordings for launch material, and benchmark generation quality. After this sprint, there are real, compelling examples showing Voce IR works — from first prompt to deployed page.
**Depends on:** Sprint 28 (voice interface complete, all generation features working)

---

## Deliverables

1. Demo 1: SaaS landing page — hero, features, pricing, testimonials, footer (minimal-saas pack)
2. Demo 2: Contact form page — form with validation, map embed, business hours, FAQ (ecommerce pack)
3. Demo 3: Multi-section marketing site — 5+ sections, animations, responsive (editorial pack)
4. Screen recordings of each conversation (text and voice sessions)
5. Benchmark results: token usage, generation time, repair cycles, Lighthouse scores
6. Quality comparison: Voce output vs v0/bolt.new equivalent prompts
7. All demo artifacts committed to `examples/demos/`

---

## Tasks

### 1. Demo 1: SaaS Landing Page (`examples/demos/saas-landing/`)

Full conversation using `voce design` with minimal-saas pack. Target sections: hero (headline, subtitle, CTA, hero image placeholder), features (6 items in 3x2 grid), pricing (3 tiers with toggle), testimonials (3 cards), footer (links, newsletter signup). Record conversation transcript. Validate: Lighthouse 95+, all a11y checks pass, < 10KB HTML.

### 2. Demo 2: Contact Form Page (`examples/demos/contact-form/`)

Conversation-driven generation focusing on forms and data. Sections: contact form (name, email, phone, message, submit with validation), embedded map placeholder, business hours table, FAQ accordion. Test incremental edit: change button color, add a field, rearrange sections. Record edit workflow.

### 3. Demo 3: Marketing Site (`examples/demos/marketing-site/`)

Complex multi-section page testing hierarchical generation. 8+ sections: navigation bar, hero with animation, about/mission, service cards, team grid, statistics counters, blog preview, newsletter CTA, footer. Use editorial pack. Test voice interface for at least 3 turns. Verify all animations compile correctly.

### 4. Screen Recordings (`docs/demos/`)

Record each demo session using terminal recording (asciinema or screen capture). Capture: the full conversation flow, plan confirmation, generation progress, compiled output preview, edit iterations. Create both full-length (5-10 min) and highlight reel (60s) versions. Export as GIF for README and MP4 for website.

### 5. Benchmarks (`docs/benchmarks/phase3.md`)

Measure and document per demo: total conversation turns, token usage (per agent breakdown), wall-clock generation time, repair cycles needed, final IR node count, compiled HTML size, Lighthouse performance/accessibility/SEO scores. Compare against: giving the same prompt to v0, bolt.new, and raw ChatGPT + manual coding.

### 6. Quality Audit (`tests/demos/quality-audit.ts`)

Automated quality checks on all 3 demos: run `voce validate` (zero errors), run `voce compile` (success), check HTML size (< 15KB), run axe-core (zero violations), check semantic structure (proper heading hierarchy, landmark regions), verify responsive meta tag and viewport settings.

### 7. Bug Fix Pass

Inevitably, running real demos will surface bugs in the pipeline. Reserve 30% of sprint capacity for fixing issues found during demo creation. Track all bugs found, fixed, and deferred. This is the real integration test.
