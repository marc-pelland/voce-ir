# Sprint 27 — MCP Server & Integrations

**Status:** Planned
**Goal:** Build a Model Context Protocol server exposing Voce IR capabilities as tools for Claude Code and other MCP clients. Add a TypeScript SDK for programmatic access and provider configuration via `voce.config.toml`. After this sprint, developers can use Voce IR directly from Claude Code or any MCP-compatible environment.
**Depends on:** Sprint 26 (incremental generation, full edit workflow)

---

## Deliverables

1. MCP server (`voce-mcp`) exposing 6 tools: validate, compile, preview, inspect, schema, examples
2. MCP resource endpoints for current IR state, brief, and decision log
3. TypeScript SDK (`@voce-ir/sdk`) for programmatic generation and compilation
4. Provider configuration via `voce.config.toml` (API keys, model selection, defaults)
5. Claude Code integration: install via `claude mcp add voce`
6. Tool documentation with usage examples

---

## Tasks

### 1. MCP Server Scaffold (`packages/mcp-server/`)

New TypeScript package implementing the MCP protocol. Use `@modelcontextprotocol/sdk` for server implementation. Entry point: `npx voce-mcp` starts the stdio transport server. Register tools and resources on initialization. Handle lifecycle: initialize, list tools, call tool, list resources, read resource.

### 2. Tool: validate (`tools/validate.ts`)

Input: IR JSON string or file path. Output: validation result with pass/fail, error list, warning list. Shells out to `voce validate --json` for structured output. Handles: inline JSON content, file paths, base64-encoded binary IR.

### 3. Tool: compile (`tools/compile.ts`)

Input: IR JSON + compile options (target: dom, output format: string/file). Output: compiled HTML string or file path. Options: minify, inline styles, include source map. Shells out to `voce compile`.

### 4. Tool: preview (`tools/preview.ts`)

Input: IR JSON or file path. Output: URL of local preview server. Starts `voce preview` in background, returns localhost URL. Includes auto-reload on IR changes. Returns server PID for cleanup.

### 5. Tools: inspect, schema, examples (`tools/inspect.ts`, `tools/schema.ts`, `tools/examples.ts`)

`inspect`: parse IR and return structured summary (node tree, type counts, validation status). `schema`: return the full Voce IR JSON schema for a given node type or the complete schema. `examples`: list available example IR files, return contents of a specific example. These are read-only informational tools.

### 6. MCP Resources (`resources/`)

Expose as MCP resources: `voce://brief` (current project brief), `voce://ir/current` (current IR state), `voce://decisions` (decision log), `voce://status` (project health). Resources update in real-time as the project evolves.

### 7. TypeScript SDK (`packages/sdk/`)

Thin programmatic wrapper: `import { VoceClient } from '@voce-ir/sdk'`. Methods: `generate(prompt, options)`, `validate(ir)`, `compile(ir, target)`, `edit(ir, change)`. Handles API key management, error handling, typed responses. Published to npm as `@voce-ir/sdk`.

### 8. Provider Config (`src/config/provider.ts`)

`voce.config.toml` at project root: API key references (env var names, not raw keys), default model, temperature, max tokens, style pack default, MCP server port. Loaded by all commands. Example config generated by `voce init`. Support for multiple providers (Claude, future: others).
